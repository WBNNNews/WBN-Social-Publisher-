How Social Publisher™ Works
One story becomes many platform-ready social posts.
The Core Idea — WBN Social Publisher™ is built on a single editorial principle: write once, publish everywhere. You write a master post in the Composer, and Social Publisher prepares it for each account and platform — respecting character limits, link formats, and media rules — so your brand voice stays consistent while the format adapts to each network.

01
Connect Your Accounts
Add your Bluesky accounts in the Accounts tab. Each account carries its own handle, default hashtags, default CTA, and time zone. OAuth authentication connects to the Bluesky AT Protocol in Phase 2.
02
Write in the Composer
Open the Composer, select a target account, and write your master post. The live character counter enforces Bluesky's 300-character limit. A real-time platform preview card shows exactly how your post will appear before you send.
03
Save, Schedule, or Publish
Save as a draft to continue later, schedule for a specific date and time, or publish immediately (live in Phase 2). Every post enters the Queue where you can track its full journey from Draft through to Published or Failed.
04
Track on the Calendar
The weekly Calendar view shows all scheduled and published posts across every connected account. Use it to spot gaps in your publishing rhythm and keep the editorial calendar full.
05
Review Analytics
Phase 1 Analytics tracks internal publishing metrics: total posts, scheduled, published, and failed broken down by account. Platform-native engagement arrives in Phase 2 with the live Bluesky connection.

Platform Architecture
Social Publisher™ is a cast application built on WBN Operating Platform™ v1.0. CSS tokens, admin tabs, cloud sync, sponsor bar, version stamp, and cast tabs are OS-level and locked. All Social Publisher-specific UI is namespaced sp- and lives in section panels only.
JSONBin Cloud Workflow
On load, the platform pulls STATE from JSONBin. Every admin save pushes the full STATE back. Check Cloud Sync Status before assuming a save is live. Falls back to local cache automatically on any sync failure. Local cache never rolls back the running version number.
Style Guide v3.2 Rules
Robin Blue is the primary OS accent — never override it with cast colours. Use --cast-hot and --cast-cold for platform-specific signals only. All JS in an IIFE. No inline onclick attribute handlers. Sponsor bar and footer stamp are mandatory on every cast.
Social Publisher™ Data Model
socialAccounts · socialDrafts · socialQueue · socialSettings · socialActivityLog. All live inside the OS STATE object and sync to JSONBin with every save.
Admin Roles
Super — full access including passwords and data reset.
Admin — marketing, sponsorship, data, OS Improvement Notes.
Editor — marketing and Social Publisher default settings only. Change all passwords before first deploy.
OS Improvement Notes
Any pattern discovered in Social Publisher™ that could benefit future WBN products should be logged here. This is the mechanism for improving WBN Operating Platform™ in parallel with the product build. Super and Admin roles can log and review notes.
Phase 2 — Live Publishing
The Publish Now button activates in v0.2 with AT Protocol OAuth. All data model structures, queue states, and composer UI are already built to support it. Phase 2 is a connection layer, not a rebuild. No structural changes to the OS layer will be required.
Daily Maintenance
Confirm Cloud Connected status on load. Review draft queue each morning. Download a data backup (Data tab) weekly. Update the Activity Log when publishing manually. Check for failed queue items and retry or remove.
